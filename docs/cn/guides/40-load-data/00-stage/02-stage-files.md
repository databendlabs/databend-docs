title: 上传到 Stage
---

import Tabs from '@theme/Tabs';

Databend 推荐两种文件上传到 Stage 的方法: [PRESIGN](/sql/sql-commands/ddl/stage/presign) 和 PUT/GET 命令。这些方法支持客户端和存储之间直接进行数据传输，消除了中间环节，并通过减少 Databend 和存储之间的流量来节省成本。

![Alt text](/img/load/staging-file.png)

PRESIGN 方法会生成一个带有签名的有时效性的 URL，客户端可以使用该 URL 安全地启动文件上传。这个 URL 授予对指定 Stage 的临时访问权限，允许客户端直接传输数据，而无需依赖 Databend 服务器进行整个过程，从而提高了安全性和效率。

如果你正在使用 [BendSQL](../../30-sql-clients/00-bendsql/index.md) 来管理 Stage 中的文件，你可以使用 PUT 命令上传文件，使用 GET 命令下载文件。

- GET 命令目前只能下载 Stage 中的所有文件，不能下载单个文件。
- 这些命令是 BendSQL 独有的，当 Databend 使用文件系统作为存储后端时，GET 命令将无法工作。

### 使用预签名 URL 上传

以下示例演示了如何使用预签名 URL 将示例文件 (books.parquet) 上传到用户 Stage、内部 Stage 和外部 Stage。

<Tabs groupId="presign">

<TabItem value="user" label="上传到用户 Stage">

```sql
PRESIGN UPLOAD @~/books.parquet;
```

结果:

```
┌────────┬────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│ Name   │ Value                                                                                                              │
├────────┼────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┤
│ method │ PUT                                                                                                                │
│ headers│ {"host":"s3.us-east-2.amazonaws.com"}                                                                              │
│ url    │ https://s3.us-east-2.amazonaws.com/databend-toronto/stage/user/root/books.parquet?X-Amz-Algorithm...               │
└────────┴────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┘
```

```shell
curl -X PUT -T books.parquet "https://s3.us-east-2.amazonaws.com/databend-toronto/stage/user/root/books.parquet?X-Amz-Algorithm=... ...
```

检查 Stage 中的文件:

```sql
LIST @~;
```

结果:

```

└───────────────┴──────┴──────────────────────────────────────┴─────────────────────────────────┴─────────┘
```

</TabItem>

### 使用 PUT 命令上传

以下示例演示了如何使用 BendSQL 将示例文件 ( books.parquet ) 上传到用户 Stage、内部 Stage 和外部 Stage。

<Tabs groupId="PUT">

<TabItem value="user" label="上传到用户 Stage">

```sql
PUT fs:///Users/eric/Documents/books.parquet @~
```

结果:

```
┌───────────────────────────────────────────────┐
│                 file                │  status │
├─────────────────────────────────────┼─────────┤
│ /Users/eric/Documents/books.parquet │ SUCCESS │
└───────────────────────────────────────────────┘
```

```sql
LIST @~;
```

结果:

```

│      name     │  size  │ ··· │     last_modified    │      creator     │
```

</TabItem>

### 使用 PUT 命令上传目录

你还可以使用带通配符的 PUT 命令从目录上传多个文件。当需要一次性暂存大量文件时，这会很有用。

```sql
PUT fs:///home/ubuntu/datas/event_data/*.parquet @your_stage;
```

结果:

```
┌───────────────────────────────────────────────────────┐
│                 file                        │status   │
├─────────────────────────────────────────────┼─────────┤
│ /home/ubuntu/datas/event_data/file1.parquet │ SUCCESS │
│ /home/ubuntu/datas/event_data/file2.parquet │ SUCCESS │
│ /home/ubuntu/datas/event_data/file3.parquet │ SUCCESS │
└───────────────────────────────────────────────────────┘
```
