## 摘要

描述了为租户创建和管理分享，并与其他租户分享这些分享。`tenant` 标识符在您的组织内唯一标识一个账户。

以下数据库对象可以被分享：

- 表
- 视图
- UDFs

所有在租户之间分享的数据库对象都是 `只读` 的，这意味着共享的对象不能被修改或删除。

## 什么是分享？

分享是封装了分享数据库所需所有信息的命名对象。每个分享包括：

- 授予访问数据库的权限以进行分享。
- 授予访问数据库中特定对象的权限。
- 与之分享数据库及其对象的消费者租户。

在从共享数据库创建数据库后，所有共享的对象都可以被消费者账户中的用户访问。

## 数据提供者和消费者

### 提供者

数据提供者是任何创建分享并使其可供其他租户消费的租户。作为数据提供者，租户可以与一个或多个租户分享数据库。对于每个共享的数据库，提供者可以使用授权来对数据库中的对象进行访问控制。

### 消费者

数据消费者是任何选择从数据提供者提供的分享中创建数据库的租户。作为数据消费者，一旦提供者将共享数据库添加到您的租户，消费者就可以访问和查询数据库中的对象，就像您会对账户中的任何其他数据库做的那样。

## 数据分享如何工作？

通过数据分享，实际上没有数据在提供者和消费者之间被复制或传输：

- 提供者在其账户中创建一个数据库的分享，并授予对数据库中特定对象的访问权限。然后，一个或多个租户（又名 `消费者`）被添加到分享中，它们可以访问共享的数据库和其他共享对象。
- 在消费者方面，从分享中创建一个 `只读` 数据库。这意味着消费者没有权限修改共享的数据对象，如 `update`、`insert`、`delete`、`drop` 等。
- 由于没有数据在提供者和消费者之间被复制，一旦提供者删除了共享的对象（如 `drop database`、`drop table` 等），那么消费者就无法再访问共享的对象了。

## 共享对象权限

共享对象权限有三种：

- USAGE：用于数据库。
- SELECT：用于表或视图。
- REFERENCE_USAGE：用于数据库。

关于权限的一些说明：

- `USAGE` 权限只能授予一个数据库的分享；然而，在该数据库内，可以将多个对象（如 UDFs、表等）的权限授予分享。
- 必须在单独的 GRANT 语句中授予单个对象的权限给分享。
- 当分享一个引用了多个数据库的对象的安全视图时，使用 `REFERENCE_USAGE` 权限，如下所示：
  - 必须单独授予每个数据库 `REFERENCE_USAGE` 权限。
  - 在授予分享对安全视图的 `SELECT` 权限之前，必须先授予数据库 `REFERENCE_USAGE` 权限。

## 分享的 DDL

为了支持创建和管理分享，Databend 提供了以下一组特殊的 DDL 命令：

- 创建分享
- 删除分享
- 修改分享
- 授予分享对象权限给分享
- 撤销分享对象权限从分享
- 显示分享
- 描述分享
- 显示授权
- 从分享创建数据库、表
- 从分享数据库表中选择数据
- 显示共享的数据库表

### 创建分享

创建一个空的新分享。一旦创建了分享，用户可以使用 `GRANT <privilege> ... TO SHARE` 命令将数据库和数据库中的对象（表）包含在分享中。然后用户可以使用 `ALTER SHARE` 来向该分享添加一个或多个租户。

语法：

```sql
CREATE SHARE [IF NOT EXISTS] <share_name> [ COMMENT = '<string_literal>' ]
```

### 删除分享

从系统中移除指定的分享，并立即撤销对所有租户的访问权限（即，创建了该分享的数据库的租户）。

语法：

```sql
DROP SHARE [IF EXISTS] <share_name>
```

### 修改分享

修改现有分享的属性。

- 从租户列表中添加或移除租户。

语法：

```sql
ALTER SHARE [IF EXISTS] <share_name> {ADD | REMOVE} TENANTS = <tenant_name> [, <tenant_name>, ...]
```

### 授予分享对象权限给分享

授予对数据库和其他支持的数据库对象（表、视图等）的访问权限给分享。授予这些对象的权限有效地将它们添加到分享中，然后可以与一个或多个租户分享。

语法：

```sql
GRANT object_privilege ON grant_share_object_name TO SHARE <share_name>

object_privilege ::= [
-- 对于 DATABASE, SCHEMA, 或 FUNCTION
    USAGE |
-- 对于 TABLE 或 VIEW
    SELECT |
-- 对于 DATABASE
    REFERENCE_USAGE]

grant_share_object_name := { DATABASE <db_name> | TABLE <db_name>.<table_name> }
```

### 撤销分享对象权限从分享

撤销对分享中的数据库和其他支持的数据库对象（表、视图等）的访问权限。撤销这些对象的权限有效地将它们从分享中移除，使它们对所有在分享中创建了数据库的租户不可访问。

语法：

```sql
REVOKE object_privilege ON grant_share_object_name FROM SHARE <share_name>

object_privilege := [USAGE|SELECT|REFERENCE_USAGE]

grant_share_object_name := { DATABASE <db_name> | TABLE <db_name>.<table_name> }
```

### 显示分享

显示您的租户创建的所有分享或您的租户可消费的分享。

语法：

```sql
SHOW SHARES;
```

输出：

`SHOW SHARES` 命令的输出是一个共享对象信息列表，包括列 `Created_on`, `Kind`, `Name`,`Database_name`, `To`, `From`, 和 `Comment`。

如果分享是由您的租户创建的，则 `Kind` 是 `OUTBOUND`；否则，如果分享可供您的租户消费，则 `Kind` 是 `INBOUND` 并且 `To` 为空。

### 描述分享

描述包含在分享中的共享对象。

语法：

```sql
{DESC|DESCRIBE} SHARE <share_name>
```

输出：

`DESCRIBE SHARE` 命令的输出是一个分享对象信息列表，包括列 `Kind`, `Name`,`Shared_on`，例如：

```sql
+----------+--------------------------------------+-------------------------------+
| Kind     | Name                                 | Shared_on                     |
|----------+--------------------------------------+-------------------------------|
| DATABASE | tenant1.db1                          | 2022-08-11 18:04:17.642 -0700 |
| TABLE    | tenant1.db1.table1                   | 2022-08-11 18:04:17.749 -0700 |
+----------+--------------------------------------+-------------------------------+
```

注意，每个分享对象的 `name` 前缀都是 `<tenant_name>.`。

- `SHOW GRANTS ON grant_share_object_name`：列出已授予对象的所有权限。
- `SHOW GRANTS OF SHARE <share_name>`：列出共享的所有租户，并指示正在使用共享的租户。

### 从共享创建数据库、表  

在租户获得访问共享对象的权限后，可以从共享中创建数据库。

语法：

```sql
CREATE DATABASE <name> FROM SHARE <provider_tenant>.<share_name>
```



### 从共享数据库表中选择数据

在租户从共享数据库创建数据库之后，租户可以像普通表一样从共享表中选择数据，在此之前，提供者必须已经授予了对共享数据库和表的访问权限。

语法：

```sql
Select * from <share_db_name>.<table_name>
```



### 显示共享数据库表

在租户从共享数据库创建数据库之后，租户可以显示来自共享数据库的表，它只输出已被授权访问的表。

 语法：

```sql
use <share_db_name>;
show tables;
```



## 使用SQL与共享的示例

使用SQL创建共享：

1. 使用 `CREATE SHARE` 命令创建一个空共享。
2. 使用 `GRANT <privilege> … TO SHARE` 命令将数据库添加到共享中，并有选择地授予对特定数据库对象（表）的访问权限。
3. 使用 `ALTER SHARE` 命令为一个或多个账户添加共享的访问权限。
4. 使用 `CREATE DATABASE ... FROM SHARE` 命令从共享中创建数据库。

### 步骤1：创建一个空共享

以下示例创建了一个名为t的空共享：

```sql
create share t;
```

### 步骤2：为数据库和对象向共享授予权限

要在共享中包含对象，需要为每个对象授予权限。授予权限时，首先授予对任何容器对象的使用权，然后再授予对容器中对象的使用权。例如，在授予对数据库中包含的任何表的访问权限之前，先授予对数据库的访问权限。

以下示例说明了向在上一步中创建的t共享授予以下对象的权限：

```sql
grant usage on database db1 to share t;

grant select on table db1.table1 to share t;
```

授予对象权限操作后，可以将租户添加到共享中。

### 步骤3：将租户添加到共享

以下示例将两个租户添加到t共享中：

```sql
alter share t add tenants=x, y;
```

在更改共享添加租户之后，现在租户x和y有权限使用共享对象创建对象（数据库，表）。

### 步骤4：从共享创建数据库

现在租户x从共享t创建数据库db：

```
CREATE DATABASE db FROM SHARE t
```
