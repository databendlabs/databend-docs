---
title: GEN_RANDOM_UUID
---

从 1.2.658 版本开始，生成基于 version 7 的随机 UUID。在此之前，此函数生成基于 version 4 的 UUID。

## 语法

```sql
GEN_RANDOM_UUID()
```

## 别名

- [UUID](uuid.md)

## 为什么使用 UUID v7？

- **基于时间的排序**：UUID v7 包含一个时间戳，允许事件或记录按照创建时间进行时间排序。当您需要跟踪操作顺序时，这尤其有用。

- **按时间顺序排序**：UUID v7 确保 UUID 按创建时间排序，这非常适合需要按时间对事件进行排序的场景，例如事件日志记录或维护审计跟踪。

## 版本信息

- 1.2.658 及更高版本：UUID 版本从 v4 升级到 v7。
- 1.2.658 之前的版本：UUID 生成基于 v4。

## 示例

在记录事件的应用程序中，维护正确的操作顺序至关重要。UUID v7 确保每个事件都按时间排序，从而可以轻松地按时间顺序跟踪操作。

```sql
-- 记录用户登录
SELECT GEN_RANDOM_UUID(), 'User logged in' AS event, CURRENT_TIMESTAMP AS event_time;

-- 记录用户进行购买
SELECT GEN_RANDOM_UUID(), 'User made a purchase' AS event, CURRENT_TIMESTAMP AS event_time;
```

这些查询的结果可能如下所示：
```sql
┌──────────────────────────────────────────────────────────────────────────────────────────┐
│           gen_random_uuid()          │         event        │         event_time         │
├──────────────────────────────────────┼──────────────────────┼────────────────────────────┤
│ 019329e6-26a2-7b01-b9f5-1c3c02600578 │ User logged in       │ 2024-11-14 08:59:29.313906 │
│ 019329e6-329e-73c3-b0a8-a413ce298607 │ User made a purchase │ 2024-11-14 08:59:32.381497 │
└──────────────────────────────────────────────────────────────────────────────────────────┘
```

请注意，`gen_random_uuid()` 值是按照事件发生的顺序生成的，从而可以轻松地维护时间顺序。
